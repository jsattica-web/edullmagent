## 용도

데이터에 근거한 정확한 차트가 아니라, 사람이 보고 분위기를 느끼는 컨셉 이미지를 만든다.
리포트 표지, 섹션 삽화, 주제를 상징하는 일러스트 등이 여기에 해당한다.
정확한 수치 비교가 목적이면 이 스킬이 아니라 `data-viz` 스킬을 로드한다.

## 차트 vs 생성 이미지 구분 기준

| 질문 | 차트(data-viz) | 생성 이미지(image-gen) |
|---|---|---|
| 정확한 수치를 읽어야 하나 | 그렇다 | 아니다 |
| 데이터가 입력으로 주어졌나 | 그렇다 | 보통 아니다 |
| 목적이 비교/추세 분석인가 | 그렇다 | 아니다(분위기/상징) |
| 예시 | 지역별 매출 막대그래프 | "성장하는 시장" 표지 일러스트 |

## 이미지 프롬프트 작성 가이드

좋은 프롬프트는 아래 요소를 구체적으로 담는다. 한 문장에 다 넣지 말고 나열한다.

- 대상: 무엇을 그리는가 (예: 상승하는 그래프를 배경으로 한 도시 스카이라인)
- 스타일: 사진풍 / 플랫 일러스트 / 수채화 / 3D 렌더 등
- 구도: 정면 / 부감 / 클로즈업, 여백, 표지면 텍스트 자리
- 색감과 분위기: 밝고 낙관적 / 차분한 톤 등
- 금지 요소: 텍스트 글자, 워터마크 등 원치 않는 것

## 실행 방법

다듬은 프롬프트로 `generate_image` 도구를 호출한다.
도구가 `outputs/generated_images/`에 저장하고 경로를 돌려주므로,
최종 응답에는 저장 경로 한 줄과 어떤 의도로 만들었는지 한 줄만 남긴다.

## 규칙

- 컨셉/표지 이미지는 반드시 `generate_image` 도구로만 만든다. matplotlib나 `python_repl`로 그리지 않는다(그건 차트용이다).
- 프롬프트는 영어/한국어 모두 가능하지만 구체적일수록 품질이 좋다.
- 표지 이미지는 글자를 이미지 안에 넣지 말고(흐릿하게 깨지기 쉽다) 여백을 비워 둔다.
- 생성은 비용이 드는 작업이다. 한 요청에 꼭 필요한 만큼만 호출한다.
- 최종 응답에는 `generate_image`가 돌려준 `outputs/generated_images/...` 경로를 그대로 명시한다.
